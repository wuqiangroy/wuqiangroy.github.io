python 3.4 的新模块
源码： Lib/pathlib.py

该模块提供了表示文件系统路径的类，语义合适，可用于不同的操作系统。Path 类分为两部分：纯路径（pure paths）和具体路径（concrete path）。pure paths 支持不带 I/O 的纯计算操作，具体路径继承了纯路径但是提供了 I/O 操作。

如果你重来没有使用过该模块，或者你不确定你的哪个类适用于你的系统，那么 Path 就是你所需要的。它实例化了一个代码运行平台的具体路径（concrete path）

在一些特殊的案例下，纯路径将非常有用。比如说：
  1、如果你想在 Unix 机器上操作 Windows 的文件路径，或者反过来。你不能够在 Unix 平台实例化 WindowsPath 来运行，但是你可以运行实例化的 PureWindowsPath
  2、你想要确认你的代码不实际访问系统 OS 而操作文件路径。这种情况下，只需要实例化一个纯类即可，这些操作不会访问任何 OS。

同样可看：PEP 428: pathlib 模块 - 面向对象的文件系统路径
同样可看：对于低级别的字符串路径操作，你也可以使用 os.path 模块

基本用法

引用主类：

>>> from pathlib import Path

子目录展开：

>>> p = Path('.')
>>> [x for x in p.iterdir() if x.is_dir()]
[PosixPath('.hg'), PosixPath('docs'), PosixPath('dist'),
PosixPath('__pycache__'), PosixPath('build')]

展开该目录下所有的 ptyhon 源文件:

>>> list(p.glob('**/*.py'))
[PosixPath('test_pathlib.py'), PosixPath('setup.py'),
PosixPath('pathlib.py'), PosixPath('docs/conf.py'),
PosixPath('build/lib/pathlib.py')]

在一个目录下导航:

>>> p = Path('/etc')
>>> q = p / 'init.d' / 'reboot'
>>> q
PosixPath('/etc/init.d/reboot')
>>> q.resolve()
PosixPath('/etc/rc.d/init.d/halt')

查询路径属性：

>>> q.exists()
True
>>> q.is_dir()
False

打开一个文件：

>>> with q.open() as f: f.readline()
...
'#!/bin/bash\n'

纯路径

纯路径对象提供了不用访问实际文件系统，处理路径的操作。有三种方式访问这些类：

class pathlib.PurePath(*pathsegments)
  一个代表了系统路径风格的泛类（实例它会创建 PurePosixPath 或 PureWindowsPath）：

  >>> PurePath('setup.py') # 在 Unix 机器上运行
  PurePosixPath('setup.py')
  每个pathsegments的元素可以是一个表示路径段的字符串，一个返回字符串的实现 os.PathLike 接口的对象，或者其他的路径对象：

  >>> PurePath('foo', 'some/path', 'bar')
  PurePosixPath('foo/some/path/bar')
  >>> PurePath(Path('foo'), Path('bar'))
  PurePosixPath('foo/bar')

  当pathsegments为空的时候，实例化的时候会被指定为当前目录：
  
  >>> PurePath()
  PurePosixPath('.')
  
  当给了几个绝对路径，那么最后一个作为锚点（模仿 os.path.join()）：
  
  >>> PurePath('/etc', '/usr', 'lib64')
  PurePosixPath('/usr/lib64')
  >>> PureWindowsPath('c:/Windows', 'd:bar')
  PureWindowsPath('d:bar')
  
  但是，在 Windows 路径里面，更改当前的根目录不会丢弃之前的设定：
  
  >>> PureWindowsPath('c:/Windows', '/Program Files')
  PureWindowsPath('c:/Program Files')
  
  伪斜线和单点会被折叠，但是双点（“..”）不会，因为这会改变符号链接前面路径的含义：
  
  >>> PurePath('foo//bar')
  PurePosixPath('foo/bar')
  >>> PurePath('foo/./bar')
  PurePosixPath('foo/bar')
  >>> PurePath('foo/../bar')
  PurePosixPath('foo/../bar')

  （如果 foo 是一个指向另一个目录的符号链接，那么 PurePosixPath('foo/../bar') 和 PurePosixPath('bar') 相等，这是错误的）
纯路径对象实现了 os.PathLike 接口，允许任何接受接口的地方使用。
  
  python 3.6 更改：新增 os.PathLike 接口支持

class pathlib.PurePosixPath(*pathsegments)

  PurePath 的子类，代表了非 - Windows 系文件系统路径：

  >>> PurePosixPath('/etc')
  PurePosixPath('/etc')

  pathsegments 和 PurePath 的参数类似。

class pathlib.PureWindowsPath(*pathsegments)

  PurePath 的子类，代表了 Windows 系文件系统路径：

  >>> PureWindowsPath('c:/Program Files/')
  PureWindowsPath('c:/Program Files')

  pathsegments 和 PurePath 的参数类似。

不管你运行的是什么操作系统，由于都不进行系统调用，所以你都可以实例这些类。

基本属性

路径是不变且可哈希的，同一类型的路径是可以进行比较和排序的。这些属性遵循不同类型的路径的折叠语义。

>>> PurePosixPath('foo') == PurePosixPath('FOO')
False
>>> PureWindowsPath('foo') == PureWindowsPath('FOO')
True
>>> PureWindowsPath('FOO') in { PureWindowsPath('foo') }
True
>>> PureWindowsPath('C:') < PureWindowsPath('d:')
True

不同类型的路径比较是不同的且无法排序：

>>> PureWindowsPath('foo') == PurePosixPath('foo')
False
>>> PureWindowsPath('foo') < PurePosixPath('foo')
Traceback (most recent call last):
File "<stdin>", line 1, in <module>
TypeError: '<' not supported between instances of 'PureWindowsPath' and 'PurePosixPath'

操作

斜线运算符可以建立子路径，和 os.path.join() 类似：

>>> p = PurePath('/etc')
>>> p
PurePosixPath('/etc')
>>> p / 'init.d' / 'apache2'
PurePosixPath('/etc/init.d/apache2')
>>> q = PurePath('bin')
>>> '/usr' / q
PurePosixPath('/usr/bin')

在接受实现 os.PathLike 的地方可以使用路径对象:

>>> import os
>>> p = PurePath('/etc')
>>> os.fspath(p)
'/etc'

路径的字符串表现形式是原始文件系统路径本身（原生形式，比如：Windows 下面使用反斜杠），你可以将文件路径作为字符串传递给任何方法。

>>> p = PurePath('/etc')
>>> str(p)
'/etc'
>>> p = PureWindowsPath('c:/Program Files')
>>> str(p)
'c:\\Program Files'

类似的，对路径调用 bytes 可以得到原始文件系统的字节对象，就像编码 os.fsencode() 一样。

>>> bytes(p)
b'/etc'

注意：仅建议在 Unix 下调用 bytes。Windows 下，unicode 是文件系统路径的规范表示。

访问各部分

要访问一个路径的各个“部分”（组件），可以使用以下属性：

PurePath.parts

返回一个由可访问路径各组件的 tuple:

>>> p = PurePath('/usr/bin/python3')
>>> p.parts
('/', 'usr', 'bin', 'python3')

>>> p = PureWindowsPath('c:/Program Files/PSF')
>>> p.parts
('c:\\', 'Program Files', 'PSF')

（注意驱动和本地根目录是如何在单个部分中重组的）

方法和属性

纯路径提供了提供了以下方法和属性：

PurePath.drive
代表了驱动盘符或名字的字符串，如果有的话：

>>> PureWindowsPath('c:/Program Files/').drive
'c:'
>>> PureWindowsPath('/Program Files/').drive
''
>>> PurePosixPath('/etc').drive
''

UNC 共享文件夹（UNC shares）也被视作驱动器：

>>> PureWindowsPath('//host/share/foo.txt').drive
'\\\\host\\share'

PurePath.root

代表了本地或全局的根目录，如果有的话：

>>> PureWindowsPath('c:/Program Files/').root
'\\'
>>> PureWindowsPath('c:Program Files/').root
''
>>> PurePosixPath('/etc').root
'/'

UNC 共享文件夹（UNC shares）永远都有根目录：

>>> PureWindowsPath('//host/share').root
'\\'

PurePath.anchor

驱动器和根目录的并存：

>>> PureWindowsPath('c:/Program Files/').anchor
'c:\\'
>>> PureWindowsPath('c:Program Files/').anchor
'c:'
>>> PurePosixPath('/etc').anchor
'/'
>>> PureWindowsPath('//host/share').anchor
'\\\\host\\share\\'

PurePath.parents

提供访问路径的逻辑祖先路径的不可变序列：

>>> p = PureWindowsPath('c:/foo/bar/setup.py')
>>> p.parents[0]
PureWindowsPath('c:/foo/bar')
>>> p.parents[1]
PureWindowsPath('c:/foo')
>>> p.parents[2]
PureWindowsPath('c:/')

PurePath.parent

路径的逻辑父路径：

>>> p = PurePosixPath('/a/b/c/d')
>>> p.parent
PurePosixPath('/a/b/c')

你不能传递一个锚点和空路径:

>>> p = PurePosixPath('/')
>>> p.parent
PurePosixPath('/')
>>> p = PurePosixPath('.')
>>> p.parent
PurePosixPath('.')

注意：这是一个纯语法操作，于是有了以下情况：

>>> p = PurePosixPath('foo/..')
>>> p.parent
PurePosixPath('foo')

如果你想向上走任意的文件系统路径，推荐先使用 Path.resolve() 解析符号链接，消除 .. 组件。

PurePath.name

代表最终路径组建的字符串，包括驱动器和根目录，如果有的话：

>>> PurePosixPath('my/library/setup.py').name
'setup.py'

UNC 驱动器没有名字：

>>> PureWindowsPath('//some/share/setup.py').name
'setup.py'
>>> PureWindowsPath('//some/share').name
''

PurePath.suffix

最终路径组件的文件的扩展名，如果有的话：

>>> PurePosixPath('my/library/setup.py').suffix
'.py'
>>> PurePosixPath('my/library.tar.gz').suffix
'.gz'
>>> PurePosixPath('my/library').suffix
''

PurePath.suffixes

列表形式的文件扩展名：

>>> PurePosixPath('my/library.tar.gar').suffixes
['.tar', '.gar']
>>> PurePosixPath('my/library.tar.gz').suffixes
['.tar', '.gz']
>>> PurePosixPath('my/library').suffixes
[]

PurePath.stem

不带后缀名的最终路径组件：

>>> PurePosixPath('my/library.tar.gz').stem
'library.tar'
>>> PurePosixPath('my/library.tar').stem
'library'
>>> PurePosixPath('my/library').stem
'library'

PurePath.as_posix()

返回带斜杠的代表路径的字符串：

>>> p = PureWindowsPath('c:\\windows')
>>> str(p)
'c:\\windows'
>>> p.as_posix()
'c:/windows'

PurePath.as_uri()

将路径表示为 URI，如果路径不是绝对路径会抛 ValueError 错误。

>>> p = PurePosixPath('/etc/passwd')
>>> p.as_uri()
'file:///etc/passwd'
>>> p = PureWindowsPath('c:/Windows')
>>> p.as_uri()
'file:///c:/Windows'

PurePath.is_absolute()

路径是否为绝对路径。一个路径同时有根和类型允许的驱动器，那么这个路径视为绝对路径：

>>> PurePosixPath('/a/b').is_absolute()
True
>>> PurePosixPath('a/b').is_absolute()
False
``
>>> PureWindowsPath('c:/a/b').is_absolute()
True
>>> PureWindowsPath('/a/b').is_absolute()
False
>>> PureWindowsPath('c:').is_absolute()
False
>>> PureWindowsPath('//some/share').is_absolute()
True

PurePath.is_reserved()

PurePath.joinpath(*other)

PurePath.match(pattern)

PurePath.relative_to(*other)

PurePath.with_name(name)

PurePath.with_suffix(suffix)
